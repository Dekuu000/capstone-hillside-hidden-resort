# Auth Sign-In Override

- Layout pattern: centered auth card + trust panel
- Primary CTA: `Sign in` at bottom of form
- Density: low, clear and calm
- Key components: `Input`, `Button`, `Toast`, `Skeleton`
- Loading behavior: disable submit, show spinner in button
- Empty behavior: not applicable
- Error behavior: field-level + form-level alert
- Offline behavior: form disabled with retry note when network unavailable
